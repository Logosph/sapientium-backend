# Instructions for coding agents

Правила проекта (список пополняется по мере необходимости).

## Правила

1. **HTTP-статусы.** Не использовать «магические числа» в коде и тестах. Для кодов ответа использовать константы из `starlette.status` (например, `status.HTTP_200_OK`, `status.HTTP_404_NOT_FOUND`).

2. **Комментарии и докстринги.** Не писать поясняющих комментариев и докстрингов. Исключение: точечные директивы mypy `# type: ignore[код]` там, где без них не обойтись.

3. **`typing.cast`.** Не использовать. Вместо `cast` подавлять соответствующие предупреждения mypy через `# type: ignore[...]` с нужным кодом ошибки.

4. **Имена.** Не начинать имена своих функций и переменных с символа `_` (ни «приватных» префиксов, ни «внутренних» соглашений Python — публичные имена без ведущего подчёркивания).

5. **Длина имён переменных.** Не использовать имена переменных из одной буквы (кроме счётчиков цикла вроде `i` в `range`, если контекст тривиален).
